# SOC Simulado - Projeto de Cibersegurança

Este repositório simula um **Security Operations Center (SOC)**, com foco em análise de logs, resposta a incidentes e aplicação de boas práticas de segurança da informação.

---

##  Objetivos
- Demonstrar como um SOC identifica e responde a incidentes.
- Simular ataques (brute force, SQL Injection) e defesas.
- Documentar processos de resposta e relatórios.
- Relacionar práticas com normas internacionais de segurança.

---

##  Estrutura do Projeto
- `scripts/` → Scripts de análise de eventos e logs.
- `docs/` → Playbooks e documentação de resposta a incidentes.
- `relatorios/` → Relatórios simulados de ataques e defesas.
- `exemplos_logs/` → Logs de exemplo para testes.

---

##  Conformidade com ISO/IEC 27001
Este projeto aplica práticas alinhadas à norma **ISO/IEC 27001**:

- **Gestão de riscos:** Identificação de vulnerabilidades simuladas.
- **Controles de segurança:** Hashing de senhas, sanitização de entradas e monitoramento contínuo.
- **Resposta a incidentes:** Playbooks documentados em `docs/`.
- **Melhoria contínua:** Relatórios em `relatorios/` para registrar ataques simulados e medidas corretivas.

---

##  Normas relacionadas
- **ISO/IEC 27001** → Gestão da Segurança da Informação.  
- **ISO/IEC 27002** → Controles de segurança recomendados.  
- **ISO/IEC 27035** → Gestão de incidentes de segurança da informação.  

---

##  Como usar
1. Execute os scripts em `scripts/` para analisar logs.  
2. Consulte `docs/` para playbooks de resposta.  
3. Leia os relatórios em `relatorios/` para entender os incidentes simulados.  
4. Compare ataques em `exemplos_logs/` com defesas implementadas.  

---

##  Benefício
Este repositório serve como **laboratório de cibersegurança**, mostrando como aplicar normas ISO em um SOC simulado e como documentar processos de segurança de forma profissional.
